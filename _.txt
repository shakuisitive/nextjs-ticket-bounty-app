LEARNING:
- When we import a server component inside a client component/page, that server component too becomes a client one. In essence, All the components are server by default unless marked with 'use client' then it becomes a client component/page and all its s descendants also become a client component.
- Through composition pattern, we can pass components as child to a component thats client component and they still wont be client. (VIDEO: 63 - client server composition) 
- Suspense only works when async operations happen inside the Suspense boundary, not before it so we must move data fetching into a separate component and then wrap that component by Suspense (hence making async operations happen inside the Suspense boundary) to trigger the fallback - (VIDEO: 67 - streaming)
- If we wanna do a page level Suspense, we can put loading.tsx at the same level as page.tsx. That loading.tsx will replace page.tsx in a way until everything is there to display in page.tsx. While with Suspense, we have more fine grained control, we can pre render other part of the apps while only waiting for the one that's not available yet. But loading.tsx will suspend everything and show loading.tsx content as a replacement of page.tsx. (VIDEO: 69.5)
- We dont get ErrorBoundary by default in NextJS that we can wrap our Suspense so in case if an async operation fails inside our Suspense, that ErrorBoundary will run. So we have to install a package called react-error-boundary and use ErrorBoundary to wrap our Suspense else it will be directed towards error.tsx which we may or may not want as that will display the whole content of error.tsx but we may want to only show Error component in area where the async operation failed, not the entire page. 
- If you have conditionaling styling to do in tailwind, use clsx instead of doing things like booleanExp && text-through as whatever method you do, you will either get false or an extra space if you're using short circuiting and ternary respectively. That will work like clsx(baseStyleInString, {conditionalTailwindStyle: true}) only if it is true will that style be applied
- Whenever we do any type changes in the future, do npm run type from package.json's script, "type": "tsc --noEmit"
- (UNABLE TO MAKE IT WORK) - If we're using let's say a component library like shadcn then if we do asChild on its button components, its gonna pass down all the classes thats applied to that button to the child that was wrapped by that Button component. If we dont wanna do that, we can also call buttonVariants() in className prop.  
- Go to shadcn and customize "theme" yourself, copy the code they give, paste it into your css and see if that works. Right now it doesnt but learn how to make it work
- It's generally a good practice to import with prefix Lucide so instead of Book, import LucideBook from lucide react as it also has Link which may collide with Next's one, so thats' why it's generally a safer bet to import it with a prefix
- We can pass suppressHydrationWarning as prop to our html in layout to get rid of hydration error
- The reason we do 'use server' in our action servers is because they can be executed on the client component too which we dont want so we say this must be executed on the server
- If you wanna use an action server on the server, dont go hard and make the entire page 'use client' for the sake of attaching an onClick on the button of that page, instead, use a form, and pass that server action on the form's action attribute.
- Export const dynamic = "force-dynamic", export const revalidate = 10 (or 0), revalidatePath(); 
- There might be a sitaution where in One UI rendering pass, we call same function with same input, in that case, we can wrap that function with cache that's given by react
- Even when we do SSG using generateStaticParams, sinec they're being cached, in case they're deleted, they will continue to be found on the url. Let's say we pre generated pages from [ticketId]/page.tsx, then visit that ticket like tickets/294u0294 and delete it. even that will be deleted from DB, if we visit it in the url like tickets/294u0294, we'll find it, even though, in reality, it has been deleted
